# Requirements: Model Failure Lab

**Defined:** 2026-04-01
**Archived:** 2026-04-02
**Core Value:** Make structured LLM failure analysis simple, reproducible, and easy to inspect from local artifacts.

## v2.1 Requirements

### Investigation Flow

- [x] **FLOW-04**: User can reload or share a run or comparison detail URL and return to the same selected lens, case, and investigation context.
- [x] **FLOW-05**: User can jump directly between major run or comparison investigation sections without long-scroll hunting.

### Cross-Artifact Drillthrough

- [x] **TRACE-01**: User can jump from a selected comparison case delta directly into that case in the baseline run.
- [x] **TRACE-02**: User can jump from a selected comparison case delta directly into that case in the candidate run.
- [x] **TRACE-03**: User can open raw-backed context for the currently selected run or comparison case without manually searching artifact identifiers.

### Detail Surface Tightening

- [x] **UIF-04**: User sees run or comparison provenance in one clear durable location instead of repeated across the detail shell.
- [x] **UIF-05**: User can immediately tell which case and investigation context are active while scanning dense evidence cards and tables.

## Deferred Follow-On Work

- **DX-02**: User can install and run `failure-lab` cleanly from a standard package install without relying on editable local development setup.
- **ADAPT-01**: User can run the same dataset-report loop through an Ollama adapter that matches the existing artifact contract.

## Out of Scope

| Feature | Reason |
|---------|--------|
| New bundled dataset packs | `v1.9` already delivered the first product-ready packs; `v2.1` should tighten the investigation loop instead of broadening content scope again |
| New classifier families or learned scoring | Keep the debugger aligned to the existing heuristic contract instead of reopening engine semantics |
| Packaging and installer work beyond minor debugger-driven fixes | Deserves its own post-`v2.1` milestone once the interaction loop is tighter |
| Hosted backends, databases, or remote artifact stores | Local JSON artifacts remain the canonical product contract |
| Additional adapter families beyond current seams | Adapter breadth should follow a stronger debugger loop, not precede it |
