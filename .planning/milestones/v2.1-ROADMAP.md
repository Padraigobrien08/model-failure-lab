## Phase Details

### Phase 64: Detail State And Section Jumps

**Goal:** Make run and comparison detail routes deep-linkable and faster to traverse.
**Requirements**: [FLOW-04, FLOW-05]
**Success Criteria**:
1. Run and comparison URLs preserve the active lens, selected case, and other investigation state that matters when a page reloads or a link is shared.
2. Users can jump directly between the major sections of a run or comparison detail page instead of long-scroll hunting.
3. Returning from related routes preserves enough state that the detail workflow still feels like one investigation, not repeated re-entry.

### Phase 65: Cross-Artifact Case Drillthrough

**Goal:** Connect comparison deltas and selected cases directly to baseline, candidate, and raw-backed evidence.
**Requirements**: [TRACE-01, TRACE-02, TRACE-03]
**Success Criteria**:
1. A selected comparison case delta can open the matching case in the baseline run with one action.
2. A selected comparison case delta can open the matching case in the candidate run with one action.
3. The currently selected run or comparison case exposes a direct handoff into raw-backed context or artifact identifiers without manual lookup.

### Phase 66: Provenance Consolidation And Flow Verification

**Goal:** Reduce duplicated detail-route chrome, strengthen active selection clarity, and verify the full investigative loop.
**Requirements**: [UIF-04, UIF-05]
**Success Criteria**:
1. Run and comparison provenance is shown once in a durable place instead of being repeated across header, shell, and side rail.
2. Active case selection is visually obvious across cards, tables, and the evidence panel.
3. The milestone closes with explicit verification of the real workflow: inventory -> comparison -> run -> selected case -> raw-backed context.
