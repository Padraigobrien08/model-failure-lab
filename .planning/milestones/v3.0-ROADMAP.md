## Phase Details

### Phase 67: Standard Package Install Path

Goal: Remove editable-checkout assumptions and make the packaged CLI work end-to-end over bundled
assets.
**Requirements**: [DX-02, DX-03]
**Plan progress**: 2/2 complete (`67-01` and `67-02` shipped)
- A fresh standard package install exposes the `failure-lab` CLI without repo-local setup hacks.
- Users can run `demo`, `run`, `report`, and `compare` from the installed package and produce
  local artifacts they can inspect by hand.
- Missing assets, path issues, or package-install failures are explicit instead of leaking
  development-only assumptions.

### Phase 68: Ollama Adapter Integration

Goal: Add Ollama model execution and CLI configuration while preserving the existing
run/report/compare artifact contract.
**Requirements**: [ADAPT-01, ADAPT-02]
**Plan progress**: 2/2 complete (`68-01` and `68-02` shipped)
- Users can point `failure-lab` at Ollama and choose a model through explicit CLI configuration.
- Ollama-backed runs persist the same core artifact shape as existing adapters.
- Reporting and comparison commands work over Ollama artifacts without adapter-specific forks in
  the pipeline.

### Phase 69: Artifact Compatibility And Distribution Verification

Goal: Prove packaged and Ollama-produced artifacts remain fully compatible with the current
debugger workflow.
**Requirements**: [ADAPT-03]
**Plan progress**: 2/2 complete (`69-01` and `69-02` shipped)
- Ollama-backed runs, reports, and comparisons open in the existing debugger surfaces without new
  adapter-specific UI paths.
- The packaged quickstart covers install through debugger inspection over real local artifacts.
- The milestone closes with explicit verification of both the packaged CLI path and the Ollama
  investigation path.
